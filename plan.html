LOGGING IN WITH FACEBOOK

Users are able to log in with their Facebook account if available.

Acceptance criteria:

  • Option to sign in with facebook
  • Logs into appropriate account given accurate credentials
  • Able to share events on facebook
  • Able to share events on facebook

Feasability: High

Dependency: Facebook API, Database

LOGGING IN AS GUEST

Users are able to use the application as a guest. This will limit the abilities the users have on the website.

Acceptance criteria:

  • Able to use site without facebook / as a guest
  • Able to view local events

Feasability: High

Dependency: -

ADDING FRIENDS OFF FACEBOOK/INVITING FRIENDS

Users can see a list of Facebook friends that use the application and those who do not. Then, they may choose to add current users, invite friends to use the application, or skip the step altogether.

Acceptance criteria:

  • Accurate list of friends using the application is accurate
  • Ability to add them
  • Accurate list of friends who do not use the application
  • Ability to invite them
  • Option to skip this step

Feasability: High

Dependency: Facebook API, Database

EVENTS WILL LIST WHICH FRIENDS ARE GOING

For each event in the Event class, we will be able to see how many friends (from Facebook) is going to the event.

Acceptance criteria:

  • Accurately gets the event and all of its data
  • Correctly identifies if a friend is going to the event
  • Adds up the total of friends going to the event
  • - Displays information such as pictures and names of friends attending

Feasability: High

Dependency: Database

EACH PERSON'S PROFILE WILL LIST THE EVENTS THEY'RE ATTENDING

Users can view what events their friends are going to on their dashboard/profile tab. This can only occur if the User and his or her friend are listed as "buddies" on LetsGo

Acceptance criteria:

  • Accurately lists the events users have RSVP’d to
  • Only able to see other users events if they are friends

Feasability: High

Dependency: Database

EVENT REEL ON HOMEPAGE IN A PARTICULAR CITY/RSVP TO EVENTS/INVITE FRIENDS

User is able to see local events within their city. Users may RSVP to these events, and may invite friends to view these events.

Acceptance criteria:

  • Accurate, up to date list of events that are in the city
  • Ability to RSVP to them / add to “basket”
  • Ability to invite friends to look at event
  • Invite is sent to the right friend, with the right event

Feasability: High

Dependency: Database

SEARCH FUNCTIONS TO CHOOSE A LOCATION AND THEME

User is able to change the location for which events are shown. User may also look for specific themes for the events.

Acceptance criteria:

  • The events shown accurately reflect the city the user has chosen
  • The events show on screen accurately reflect the theme the user has searched for

Feasability: High

Dependency: Eventbrite API, Google events API

NOTIFICATIONS

User gets notifications of events that their friends have invited you to view

Acceptance criteria:

  • User is notified if a friend invites them to view an event
  • The notification is to the right event

Feasability: High

Dependency: Database
